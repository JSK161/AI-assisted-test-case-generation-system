# AI辅助测试用例生成系统测试计划说明书

## 1. 文档概述

本文档用于说明 AI辅助测试用例生成系统基础层的测试目标、测试范围、测试方法、测试环境、测试用例设计和验收标准。测试重点是验证系统能否完成登录、项目管理、需求录入、AI生成测试用例、测试用例管理和采纳率统计等核心流程。

## 2. 测试目标

基础层测试目标如下：

1. 验证系统主要功能是否符合需求规格说明书。
2. 验证前后端接口是否能够正常交互。
3. 验证 MySQL 数据持久化是否正确。
4. 验证 DeepSeek API 是否能够生成结构化测试用例。
5. 验证异常场景下系统是否给出明确提示。
6. 验证系统是否满足专业实训基础层验收要求。

## 3. 测试范围

### 3.1 纳入测试范围

1. 用户登录。
2. 项目创建和查询。
3. 需求录入和查询。
4. DeepSeek API 测试用例生成。
5. 测试用例保存、查询和编辑。
6. 测试用例采纳状态维护。
7. 采纳率统计。
8. 基础异常提示。
9. README 部署流程验证。

### 3.2 不纳入基础层测试范围

1. 知识库/RAG。
2. 多模型切换。
3. 流式输出。
4. 多角色复杂权限。
5. 高并发性能测试。
6. 自动化测试脚本生成。

## 5. 测试方法

基础层采用以下测试方法：

1. 功能测试：验证页面功能和接口功能是否正确。
2. 接口测试：使用浏览器、前端页面或接口工具验证 REST API。
3. 数据库测试：检查数据是否正确保存到 MySQL。
4. 异常测试：输入非法数据或断开 DeepSeek API 配置，验证错误提示。
5. 集成测试：验证从登录到统计的完整业务流程。

## 7. 接口测试计划

### 7.1 登录接口

接口：

```text
POST /api/auth/login
```

测试点：

1. 正确账号密码返回成功。
2. 错误密码返回失败。
3. 空账号或空密码返回参数错误。

### 7.2 项目接口

接口：

```text
POST /api/projects
GET  /api/projects
GET  /api/projects/{id}
```

测试点：

1. 创建项目成功。
2. 项目名称为空时返回错误。
3. 项目列表返回数据结构正确。
4. 查询不存在项目时返回错误提示。

### 7.3 AI生成接口

接口：

```text
POST /api/ai/generate-test-cases
```

测试点：

1. 有效需求可以生成测试用例。
2. requirementId 不存在时返回错误。
3. DeepSeek API 调用失败时返回错误。
4. 成功生成后 test_case 表和 ai_generation_record 表均有记录。

## 8. 数据库测试计划

数据库测试重点如下：

1. 登录用户是否存在于 `sys_user` 表。
2. 项目创建后是否写入 `project` 表。
3. 需求录入后是否写入 `requirement` 表。
4. AI生成测试用例后是否写入 `test_case` 表。
5. 每次 AI调用是否写入 `ai_generation_record` 表。
6. 修改采纳状态后 `test_case.adoption_status` 是否更新。
7. 删除或查询不存在数据时是否有合理提示。

## 9. 异常测试计划

| 异常场景 | 操作方式 | 预期结果 |
| --- | --- | --- |
| 登录密码错误 | 输入错误密码 | 提示账号或密码错误 |
| 项目名称为空 | 创建项目时不填名称 | 提示项目名称不能为空 |
| 需求描述为空 | 提交空需求 | 提示需求描述不能为空 |
| DeepSeek API Key 缺失 | 清空 API Key 后生成 | 提示 AI 服务配置异常 |
| AI返回格式错误 | 模拟非 JSON 返回 | 提示 AI 返回格式不正确 |
| 数据库连接失败 | 停止 MySQL 后请求接口 | 后端返回服务异常提示 |

## 10. 集成测试流程

完整集成测试流程如下：

1. 启动 MySQL。
2. 启动 Spring Boot 后端。
3. 启动 Vue 前端。
4. 使用 admin/123456 登录。
5. 创建项目。
6. 录入用户登录功能需求。
7. 点击 AI生成测试用例。
8. 查看测试用例列表。
9. 修改其中一条测试用例。
10. 将部分测试用例标记为已采纳。
11. 进入统计页查看采纳率。
12. 检查 MySQL 中的数据记录。

预期结果：

1. 全流程无阻塞错误。
2. AI生成结果格式清晰。
3. 数据能够正确保存和查询。
4. 采纳率统计与实际数据一致。

## 11. 验收标准

基础层测试通过标准如下：

1. 登录、项目管理、需求录入、AI生成、用例管理和统计功能均可正常使用。
2. 主要接口返回结构统一。
3. 核心数据可以持久化到 MySQL。
4. DeepSeek API Key 不出现在前端代码中。
5. 常见异常有明确提示。
6. README 可以指导他人在本地启动系统。
7. 完成至少一轮完整集成测试并记录测试结果。
